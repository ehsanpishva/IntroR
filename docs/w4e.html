Excercise 4a

For this exercise, you will need this dataset:

  • mirnadata.rdata
  1. The code for a differential miRNA expression analysis of Alzheimers disease in post-mortem brains is pasted below (remember to set the working directory before trying to load the dataset).
load("mirnadata.rdata")

res <- matrix(data=NA, nrow=490, ncol=4)

for (i in 2:491){
   fit <- lm(dat[,i] ~ Braak, data=dat)
   res[i-1,] <- coef(summary(fit))[2,]
}

colnames(res) <- c("Estimate","SE","t","p-value")
rownames(res) <- colnames(dat)[2:491]
res <- res[order(res[,4]),]
  1. Install the package multiMiR with:
install.packages("BiocManager")
BiocManager::install("multiMiR")

Then load the package with:

library(multiMiR)

  1. Use the get_multimir function from the package and extract the validated mRNA targets for the top (i.e., most significant) two deferentially expressed miRNAs in association with AD pathology (read help(get_multimir) carefully to check which argument of the function you have to use to specify the two miRNAs and also carefully check how the names of the two miRNAs need to be specified so that they can be looked up). Put the results that the function returns into an object called top2.

  2. Generate a vector from the column called target_entrez in the table of results returned by the get_multimir function. You can do this with:

targets <- as.data.frame(top2@data)$target_entrez
  1. Install packages clusterProfiler and org.Hs.eg.db with:
BiocManager::install("clusterProfiler")
BiocManager::install("org.Hs.eg.db")

Then load the clusterProfiler package with:

library(clusterProfiler)

  1. Use the enrichKEGG function from the clusterProfiler package and use the targets vector you generated above as input.

  2. Write the table of the KEGG pathway analysis to a file (use the write.table function; see the week 1 extra part) and upload that together with your script.

Excercise 4b

For this exercise, you will need this dataset:

  • data_bdi.dat

The dataset includes the responses of 276 patients with obsessive-compulsive disorder (OCD) to the 21 items on the Beck Depression Inventory. The patients were assessed twice, with a one-year delay between the two assessments.

The items on the BDI measure the following indicators of depression:

  1)  Sadness
  2)  Pessimism
  3)  Sense of Failure
  4)  Dissatisfaction
  5)  Guilt
  6)  Sense of Punishment
  7)  Dislike of Self
  8)  Self Accusation
  9)  Suicidal Ideation
  10) Episodes of Crying
  11) Irritability
  12) Social Withdrawal
  13) Indecisiveness
  14) Negative Body Image
  15) Retardation / Work Inhibition
  16) Insomnia
  17) Fatigability
  18) Loss of Appetite
  19) Loss of Weight
  20) Somatic Preoccupation
  21) Loss of Libido

  1. Read in the dataset using the read.table function.

  2. Create a vector with 21 missing values with:

cors <- rep(NA, 21)

  1. For each of the 21 items, correlate the pre- and post-assessment scores. Use a loop to do this. Save the results into the cors vector.

  2. Which item correlates the weakest? Which item correlates the strongest? What is the average correlation?

  3. Let’s see if the 21 items can be clustered into different groups based on their pre to post correlations. Install and load the mclust package with:

install.packages("mclust")
library(mclust)

  1. Use the Mclust function from the package to run a cluster analysis (using a mixture model) with the correlations as input. Specify that you want 2 clusters. For the model type (modelNames), specify that you want type "E" (which asumes that the variance of the correlations within clusters is the same for the 2 clusters).

  2. Which items cluster together into the first and second groups?

  3. Repeat step 6. but now specify that you want between 1 and 6 clusters. The Mclust function will determine the optimal number of clusters according to the BIC (here, a higher value of the BIC indicates a better tradeoff between the fit of the mixture model and its complexity). What number of clusters has the highest BIC? Create a plot of the BIC values (read help(plot.Mclust) to see how this can be done).
